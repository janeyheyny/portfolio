# Commit Naming

This project follows a lightweight Conventional Commits style to keep the history clear and meaningful:

- `feat:` new features
- `fix:` bug fixes
- `style:` visual or formatting changes
- `refactor:` internal code improvements
- `docs:` documentation updates
- `chore:` maintenance & tooling

I aim for small, intentional commits so changes are easy to understand. Please see [Conventional Commits](https://www.conventionalcommits.org/) for more.
